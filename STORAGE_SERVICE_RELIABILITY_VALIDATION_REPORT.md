## Architecture Overview

### Storage Architecture
The Storage Service uses a **MinIO (MinIO) S3-compatible object storage** solution for file storage, with CockroachDB for file metadata and user quota tracking.

**Components:**
1. **MinIO Client** - Object storage for presigned URLs
2. **CockroachDB** - File metadata and user quota tracking
3. **Presigned URLs** - Time-limited URLs for file access (15 minutes)
4. **File Repository** - CRUD operations for file metadata

**Upload Flow:**
1. Client initiates upload via HTTP API (`GenerateUploadURL`)
2. Service validates storage quota
3. Service generates file ID and MinIO object key
4. Service creates presigned URL (valid for 15 minutes)
5. Service saves file metadata to CockroachDB with status "uploading"
6. Client uploads file to MinIO using presigned URL
7. Client calls `CompleteUpload` to mark file as completed
8. Service updates file status to "completed"

**Download Flow:**
1. Client requests download via HTTP API (`GenerateDownloadURL`)
2. Service validates file ownership
3. Service generates presigned download URL (valid for 1 hour)
4. Client downloads file from MinIO using presigned URL

## Critical Reliability Issues (CRITICAL)

### ðŸš¨ **CRITICAL #1: No File Size Validation on Upload**

**Reliability Impact:** HIGH  
**Exploitability:** HIGH - Could lead to resource exhaustion and abuse  
**Location:** [`secureconnect-backend/internal/handler/http/storage/handler.go:26-31`](secureconnect-backend/internal/handler/http/storage/handler.go:26-31)

**Issue Description:**
The `GenerateUploadURLRequest` struct has `FileSize` validation but does NOT enforce the `MaxAttachmentSize` constant (50MB). This could lead to:
- Resource exhaustion (MinIO storage costs)
- Abuse (oversized file uploads)
- Database bloat (large file metadata records)
- Performance degradation

**Reproduction Scenario:**
1. Attacker initiates a file upload with `FileSize: 10GB` (10,737,418,176 bytes)
2. Handler validates `FileSize` is >= 1 (passes validation)
3. Service accepts the upload and generates a presigned URL
4. Attacker uploads 10GB file to MinIO
5. MinIO storage costs are consumed
6. Database record is created with 10GB file size
7. Service resources are overwhelmed

**Current Code:**
```go
// GenerateUploadURLRequest represents upload URL request
type GenerateUploadURLRequest struct {
    FileName    string `json:"file_name" binding:"required"`
    FileSize    int64  `json:"file_size" binding:"required,min=1"`  // No max validation!
    ContentType string `json:"content_type" binding:"required"`
    IsEncrypted bool   `json:"is_encrypted"`
}

// In handler/http/storage/handler.go
func (h *Handler) GenerateUploadURL(c *gin.Context) {
    var req GenerateUploadURLRequest
    if err := c.ShouldBindJSON(&req); err != nil {
        response.ValidationError(c, err.Error())
        return
    }

    // Call service
    output, err := h.storageService.GenerateUploadURL(c.Request.Context(), userID, &storage.GenerateUploadURLInput{
        FileName:    req.FileName,
        FileSize:    req.FileSize,  // NO MAX VALIDATION!
        ContentType: req.ContentType,
        IsEncrypted: req.IsEncrypted,
    })

    if err != nil {
        response.InternalError(c, "Failed to generate upload URL")
        return
    }

    response.Success(c, http.StatusOK, output)
}
```

**SAFE FIX Proposal:**
Add file size validation and enforce `MaxAttachmentSize` constant:


### ðŸš¨ **CRITICAL #2: No MIME Type Validation on Upload**

**Reliability Impact:** HIGH  
**Exploitability:** HIGH - Could lead to malicious file uploads  
**Location:** [`secureconnect-backend/internal/handler/http/storage/handler.go:26-31`](secureconnect-backend/internal/handler/http/storage/handler.go:26-31)

**Issue Description:**
The `GenerateUploadURLRequest` struct has `ContentType` validation but does NOT validate against an allowlist. This could lead to:
- Malicious file uploads (executables, scripts, etc.)
- MIME spoofing attacks
- Security vulnerabilities

**Reproduction Scenario:**
1. Attacker uploads a file with `ContentType: "application/x-sh"`
2. File is executed when downloaded
3. Other users' devices are compromised

**Current Code:**
```go
// GenerateUploadURLRequest represents upload URL request
type GenerateUploadURLRequest struct {

    IsEncrypted bool   `json:"is_encrypted"`

```

**SAFE FIX Proposal:**
Add MIME type allowlist validation:

```go

        "text/javascript":              true,

        "application/xml":                true,
        "application/pdf":                 true,

        "image/webp":                    true,
        "video/mp4":                     true,
        "video/webm":                     true,
        "audio/mpeg":                    true,
        "audio/mp3":                     true,
        "application/zip":                 true,
        "application/x-rar-compressed":    true,
        "application/x-tar":                true,
        "application/x-gzip":                true,
        "application/x-7z":                 true,
        "application/x-zip":                 true,
        "application/octet-stream":          true,
    }
)

// In handler/http/storage/handler.go
func (h *Handler) GenerateUploadURL(c *gin.Context) {
    var req GenerateUploadURLRequest
    if err := c.ShouldBindJSON(&req); err != nil {
        response.ValidationError(c, err.Error())
        return
    }

    // Validate MIME type (NEW)
    if !constants.AllowedMIMETypes[req.ContentType] {
        response.ValidationError(c, "Invalid content type: "+req.ContentType)
        return
    }

    // Call service
    output, err := h.storageService.GenerateUploadURL(c.Request.Context(), userID, &storage.GenerateUploadURLInput{
        FileName:    req.FileName,
        FileSize:    req.FileSize,
        ContentType: req.ContentType,
        IsEncrypted: req.IsEncrypted,
    })

    if err != nil {
        response.InternalError(c, "Failed to generate upload URL")
        return
    }

    response.Success(c, http.StatusOK, output)
}
```

**Required Changes:**
1. Add `AllowedMIMETypes` constant to `constants/constants.go`
2. Add MIME type validation to `GenerateUploadURL` handler
3. Add metrics for rejected uploads due to invalid MIME type

**Backward Compatibility Assessment:** âœ… SAFE

- Only adds MIME type validation to prevent abuse

- Prevents malicious file uploads
